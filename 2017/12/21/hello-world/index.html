语法

  • 目前对于各个编辑器解析 Markdown 语法的结果不尽相同,也就是说没有完全统一标准化。但基本语法基本一致，而且对于文本编辑也够用了。
    • 本文由于是用 Markdown 编辑器来介绍 Markdown 语法,最好是能通过预览来观察语法之间的差异。
    • 本文主要是讲解有道云笔记里的 Markdown 语法。

1、不是简单的纯文本

1.1、标题（Heading）

1.1.1、语法
  • 用 ++“#”++ 代表标题层级,h1~h6 分别用 1 个 ++“#”++ ~ 6 个 ++“#”++
1.1.2、应用
  • 例如: h3 可以表示为 ++“### XXX”++; h5 可以表示为 ++“##### XXX”++

注意：

  • “#” 后必须加空格
  • h1 和 h2 默认有条灰色横线

1.2、加粗（Bold）

1.2.1、语法
  • 用 4 个 “*” 表示「加粗」
1.2.2、应用
  • 正常文本 加粗文本

注意:

  • 用语法标记的文字内部左右不加空格,符号外部最好加上空格，不加可能导致语法不生效或者语法错乱

2、样式属性（Style）

2.1、水平分割线（Horizontal rule）

2.1.1、语法
  • 用 3 个 “-“表示「水平分割线」
2.1.2、应用
  • 一般用来分割 title 和 content ,例如:本文所示

2.2、引用（Quote）

2.2.1、语法
  • 用 “> “ 表示「引用」
2.2.2、应用
  • 「引用」所呈现的文本样式个人表示很喜欢,可以用来编写 总结性/提示性 文本

注意:

  • 如果连续的两行(中间没有空行)使用,则两行「引用」同行展示,且不换行
  • 如果要换行,可以使用空行来分隔,但「引用」则不连续
  • 若要既连续又换行,则可以但不仅限于和「无序列表」/「有序列表」联合使用
    • 联合使用时,可用 “>-/ “ / “> -/ “ 表示
    • 个人认为用后面空格隔开的那种,第一可以让语法统一格式标准化,第二个人感觉空格隔开来让 Markdown 源码看起来更美观清爽一些

2.4、有序列表（Ordered list）

2.4.1、语法
  • 用 “数字. “ 表示 「有序列表」
2.4.2、应用
  1. 一级
    1. 两级
    1. 两级
      1. 三级
        1. 四级
          1. 五级
  • 本人一般情况不太使用「有序列表」,除非是那种明确表示步骤/层级优先级的文本结构

总结:

  • 第一级与数字大小无关,有多少行,一级的数字为多少
  • 二级或三级都跟设置的数字有关,二级用阿拉伯数字表示,三级用小写英文字母表示
  • 三级或三级以上均用小写的英文字母表示,超过 26 的部分字母叠加使用

3、高级功能（Advanced function）

3.1、超链接（Link）

3.1.1、语法
  • 用 “[超链接文本]”+”(超链接地址)” 表示「超链接」
3.1.2、应用
  • 官网链接

注意：

  • [] 和 () 之间没有空格
  • [] 中不能为空，不然没有锚点入口

3.2、图片（Image）

3.2.1、语法
  • 在「超链接」语法前 + “!” 表示「图片」
3.2.2、应用

注意:

  • “()” 中填写的图片路径必须是公网能访问到的图片路径
  • “[]” 中可不需要填写文本,其作用是在图片加载失败时起占位提醒作用,相当于 img 标签的 alt 属性
  • 这样插入的图片相对于代码块里 img 标签自定义效果不好,但对于布局上要求不高倒是方便的很
